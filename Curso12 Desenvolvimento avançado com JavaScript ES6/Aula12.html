Tipos de Testes em TI

Testes Automatizados

  • Testes Unitários

    testam a menor parte de um código: uma função, um método, uma classe

  • Testes Integrados

    testam a integração das pequenas partes testadas nos testes unitários

    tem o objetivo de integrar as partes de um mesmo código e não com outros sistemas e outros códigos externos

  • Testes Funcionais

    testam a integração de seu sistema com outros sistemas externos: testam uma funcionalidade de negócio do começo ao fim

    são chamados de etestes Black Box: como um usuário faria sem se importar com o código que está por trás

TDD: Test Driven Development

Desenvolvimento orientado a testes

É um dos pilares do Extreme Programming. Consiste em testar e Refatorar em pequenos ciclos, onde você escreve o teste antes do código, faz o mesmo passar e refatora o código.

Primeiro se escreve o teste e somente depois se escreve o código

É um ciclo de 3 etapas: i) escrita do teste; ii) escrita do código iii) Refatoração

O ciclo se repete ao longo do desenvolvimento

    Vantagens de Aplicar o TDD no seu dia-a-dia de programação
  • Feedback rápido:

    se você tem o hábito de primeiro escrever testes para depois escrever o código, então você vai garantindo que cada pequena parte do seu código está sendo testada enquanto vc vai desenvolvendo

  • Maior segurança em alterações e novas funcionalidades

    um codigo com testes permite achar mais facilmente seus erros e corrigí-los, além disso ao fazer alterações permite ver quais partes foram impactadas o que gera mais segurança no desenvolvimento de alterações e implementar novas funcionalidades

  • Código mais limpo

    fazer pequenas partes e torná-las testáveis produz um código mais limpo

  • Produtividade

    código melhor e mais enxuto, segurança para fazer alterações, tem feedback dos testes durante o desenvolvimento, a sua produtividade deveria aumentar

    na prática perde-se muito tempo corrigindo bug e dando manutenção em código, por isso, o teste não é perda de tempo. Ele sepaga sozinho na economia de tempo para achar erros. No final das contas gasta-se mais tempo para desenvolver com testes mas economiza-se tempo na hora de corrigir os bugs

    desse modo, código com teste é código com qualidade

BDD: Behavior Driven Development

Desenvolvimento orientado a comportamento

Técnica de desenvolvimento ágil que visa integrar regras de negócio com linguagem de programação

Como fazer isso ? através de ferramentas de testes como Mocha, visa escrever o teste e descrever o que o teste faz: permite descrever a funcionalidade de negócio que vc está testando

Dessem modo, o código começa a ficar documentado o que permite saber qual o comportamento esperado de cada trecho do código.

Pilares do BDD

  • Documentação
